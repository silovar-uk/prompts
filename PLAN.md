# スマホファースト・プロンプトランチャー 実装計画書

- 対象リポジトリ: `silovar-uk/prompts`
- 最終更新: 2026-07-13
- 正本: 発注者が確定したマスタープラン。矛盾する場合、本書の方針を優先する。

## 結論

作るものは「プロンプト集サイト」ではなく、スマホのホーム画面から2タップでAIに仕事を渡せるプロンプトランチャーPWA。

変更してはいけない骨格:

- 300件を一覧で見せず、候補は原則3〜5件
- 基本プロンプトとモディファイアーを組み合わせ、条件違いの複製を避ける
- 検索対象はタイトルだけでなく、検索想定文・類義語・入力物などメタデータ全体
- 検索結果ゼロで行き止まりを作らない
- プロンプト詳細は記事ではなく入力フォーム
- 貼り付けた本文・業務データを自動保存しない
- 必須入力は最大2件、任意入力は最大4件
- スマートフォンを第一に設計する

## 技術決定

- 静的サイト + PWA、バックエンドなし
- Vite + React 18 + TypeScript strict + Tailwind CSS
- Zustandによる個人設定の端末内保存
- `data/prompts/` と `data/modifiers/` は1件1JSON
- Zodを単一情報源としてJSON Schemaも生成
- ビルド時に全JSONを検証して `public/catalog.json` に結合
- 検索はNFKC正規化、カタカナ→ひらがな、類義語、bigram、フィールド重みで実装
- 検索品質は `eval/queries.json` による回帰テスト
- Vitest + Testing Library + Playwright
- GitHub ActionsからGitHub Pagesへ公開

## 合成仕様

最終プロンプトは次の順で構築する。

1. 入力変数を解決した基本プロンプト本文
2. 選択されたモディファイアーを `stance → scope → audience → output → process` の順で連結
3. 同一slot内は選択順

競合するモディファイアーは、後から選んだ方を残す。

## モバイル技術要件

- `100dvh` と `env(safe-area-inset-*)` を使用
- 最小タップ領域44×44px
- 390pxと430pxで横スクロールなし
- コントラスト比4.5:1以上、フォーカス可視
- `prefers-reduced-motion` に対応
- Service Workerでアセットとカタログをプリキャッシュ
- iOSクリップボードはユーザージェスチャー内で実行する
- JSバンドル150KB gzip以内を目安にする

### Phase A2

お気に入り、履歴、前回設定、長押しコピー、PWA磨き込み、追加ウィザード、エクスポート・インポート。

### Phase A3

### Phase A4/B8

100件から300件へ拡張し、検索辞書と重みを継続調整する。
