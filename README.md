# Prompt Launcher

文章、会議メモ、URL、コードなどを貼ると、いま使うべきプロンプトを3件に絞るスマホ向けPWAです。

設計上の正本は [`PLAN.md`](./PLAN.md) です。正式プロンプトは `data/prompts/` の1件1JSONとして管理し、ビルド時にZodで検証して `public/catalog.json` へ結合します。

## 主要画面

### ホーム

文章、メモ、URL、コード、数値データをそのまま貼り付けます。端末内の簡易判定で入力物を分類し、検索メタデータと合わせて候補を3件に絞ります。入力内容は自動保存しません。

### ライブラリ

お気に入りと端末内の自作プロンプトを管理します。自作プロンプトの追加、編集、複製、並べ替え、アーカイブ、正式版への移行もここから行います。

### 履歴

貼り付けた本文ではなく、使ったプロンプトと安全な選択条件だけを記録します。同じ条件で再開できます。

### 設定

テーマ、バックアップ、データ削除、Service Workerを含む最新版の再読み込みを管理します。

## 正式プロンプト30件

次の領域を収録しています。

- 文章・リライト：6件
- 会議・議事録：4件
- 企画・思考：5件
- 調査・比較：4件
- 広報・PR：4件
- データ整理：2件
- Web・コード：2件
- 資料・画像：2件
- 学習・解説：1件

追加27件は `scripts/generate-core-prompts.ts` から生成し、既存の3件と合わせてカタログ化します。

## 検索品質

`npm run test:search` は、30件の各プロンプトに登録された自然な検索想定文から100件の評価セットを生成し、上位3件・上位5件への到達率を確認します。

検索では次を利用します。

- NFKC正規化
- カタカナからひらがなへの変換
- 類義語辞書
- bigram一致
- 検索想定文の完全一致
- タイトル、概要、タグの重み
- お気に入り、使用回数、入力物、選択目的

## 開発

```bash
npm install --no-audit --no-fund
npm run dev
```

主なコマンド：

```bash
npm run generate:core  # 追加27件の正式プロンプトを生成
npm run catalog        # 全データを検証してcatalog.jsonを生成
npm run validate:data  # JSON、参照先、入力数、テンプレート変数を検証
npm run test:search    # 100件の検索回帰評価
npm run test           # ユニットテスト
npm run build          # 本番ビルド
npm run test:e2e       # 390px・430pxのモバイルE2E
npm run ci             # データ検証、検索、テスト、ビルドを一括実行
```

## データの安全性

貼り付けた本文や業務データは自動保存しません。端末内へ保存するのは、お気に入り、利用履歴、選択式の前回設定、自作プロンプト、アーカイブ・移行記録などです。
