# Prompt Launcher

スマホのホーム画面から、いま必要なプロンプトをすぐ呼び出すためのPWAです。

このリポジトリでは、正式プロンプトを「1ファイル＝1件」のJSONとして管理し、ビルド時にZodで検証して `public/catalog.json` へ結合します。設計上の正本は [`PLAN.md`](./PLAN.md) です。

## 開発

```bash
npm install --no-audit --no-fund
npm run dev
```

主なコマンド：

```bash
npm run validate:data  # JSON検証とcatalog生成
npm run test:search    # 検索評価
npm run test           # ユニットテスト
npm run build          # 本番ビルド
npm run test:e2e       # モバイルE2E
npm run new:prompt     # プロンプト雛形を対話生成
```

## プロンプトを追加・育成する

### スマホから試す

1. アプリの「設定」を開く
2. 「自分用プロンプトを追加」を押す
3. 名前、用途、AIへの指示、入力材料を3ステップで入力する
4. 似ている既存プロンプトを確認する
5. 「保存して使う」を押す

この段階では端末内だけに保存され、通常のプロンプトと同じように検索・お気に入り・履歴で使えます。貼り付ける本文は保存されません。

### 編集・複製する

端末内プロンプトには次の操作があります。

- `編集`：同じIDを保ったまま内容を更新する。並び順は維持される
- `複製`：別IDを発行し、元の内容を土台に新しい用途へ分岐する
- `確認`：生成される入力画面とプロンプトを確認する
- `削除`：関連するお気に入り・履歴・使用回数も一緒に整理する

編集では自分自身を重複候補から除外します。複製では複製元も候補に残し、差分が十分あるか確認できます。

### 並べ替え・アーカイブする

画面右下の `整理` を開くと、自作プロンプト専用の管理画面へ移動します。

- 上下ボタン：1件ずつ順番を微調整する
- 使用回数順：よく使うものを上へ集める
- 更新日順：最近磨いたものを上へ集める
- 名前順：一覧を五十音順に整える
- アーカイブ：検索やお気に入り候補から一時的に外し、履歴や使用回数は保持する
- 利用中へ戻す：アーカイブした内容と記録を復元する
- 完全に削除：端末内版と関連する個人データを削除する

アーカイブ済みプロンプトは通常検索から外れます。削除せずに棚上げしたい場合に使います。

### 重複を整理する

設定画面の「似ているプロンプト」には、自作プロンプトを含む近い組み合わせを表示します。

判定には次を使います。

- 名前
- 用途説明
- 検索想定文
- AIへの指示内容
- カテゴリ、目的、入力材料

一致度だけで自動削除や保存禁止にはしません。用途が同じなら編集で統合し、用途が違うなら名前や検索語に差分を出します。

### 正式カタログへ登録する

端末内プロンプトの管理画面から、次のどちらかを使います。

- `JSON`：正式登録用のJSONをクリップボードへコピー
- `GitHub ↗`：ファイル名とJSONを入れたGitHubの新規ファイル作成画面を開く

正式登録時はCIでスキーマ、参照先、入力数、検索文数を検証します。

### 正式登録済みプロンプトへ移行する

正式版の公開後は、右下の `整理` から `正式版へ移行` を選びます。

1. 自作プロンプトに近い正式版候補を確認する
2. 必要に応じて名前・用途・IDで検索する
3. 移行先を選択する

移行時には次の個人データを正式版IDへまとめます。

- お気に入り
- 使用回数と最終使用日時
- 利用履歴
- 前回使ったモディファイアーと詳細設定

端末内版は通常検索から外れ、`移行済み` に対応関係が残ります。端末内版は後から復元できますが、正式版へ移した履歴は、移行後の利用記録と混ざる可能性があるため自動では巻き戻しません。

### PCから直接追加する

1. `npm run new:prompt` で雛形を作る、または `data/prompts/` にJSONを追加する
2. ファイル名と `id` を一致させる（例：`writing-004.json`）
3. `searchPhrases` を5〜10件入れる
4. 必須入力は2件以内、任意入力は4件以内にする
5. `npm run validate:data` を実行する

モディファイアーは `data/modifiers/` に追加します。スキーマの参照用ファイルは `data/schema/` に自動生成されます。

## データの安全性

貼り付けた本文や業務データは自動保存しません。端末内に保存するのは、お気に入り、利用履歴、選択式の前回設定、自作プロンプトなどです。

通常の個人データは設定画面からJSONとしてバックアップできます。アーカイブ・移行記録は `整理` 画面の「管理データを書き出す」から別JSONとして保存できます。個人データを削除すると、アーカイブ・移行記録もまとめて消去します。
